總覽 Overview

目標

我們在使用政大圖書館場地管理系統預約場地時，常常不能順利預約到想要的空間，因此希望改善預約流程，幫助政大學生都能順利預約到想要的空間，其中討論室是同學們最常使用的空間，因此選擇以討論室預約流程，優先進行網站的再設計專案。

需求

  • 了解目前討論室預約系統行為
  • 拆解行為中未被滿足的需求
  • 重新設計資訊架構與預約流程
  • 重新打造系統品牌形象
    •

    原型

    以預約討論室為第一優化流程，篩選空間種類時，請選擇「討論室」，而可順利進入下個流程。

    問題 Problem

    透過自身操作經驗與問卷收集，了解政大學生操作預約系統感受，發現對預約流程的不滿意。

    政大學生預約討論室主要用以討論小組報告、個人自習使用，且多數使用網頁版操作預約，目的是為了想要預約到可用的空間。
    1. 切換查詢可預約討論室時很不方便，需要跳回起始頁
    2. 想要知道哪些時間空間有空討論室，需要一個一個查
    3. 網站中很難直接找到相關的場地空間資訊
    4. 預約網站的風格設計與其他圖書館網頁大相徑庭

    設計過程 Process

    01 了解預約行為

    訪談：深入了解預約需求

    訪談三位曾使用政大場地管理系統預約討論室的學生，了解預約經驗與行為動機。

    將訪談重要資訊抓取成重點便利貼，貼在 FigJam，並將相類似行為以橫列聚集，且總結成洞見（最左方不同顏色便利貼）。

    聚焦預約流程，分為預約前、中、後，並將上步驟訪談洞見整理於流程，以直欄統整成行為洞見（最上層淺灰色）。顯示使用者會接觸到介面的時機為「預約中」。

    根據「預約中」的行為洞見，以動詞名詞分析使用者行為背後的需求與期待，並進行優先排序，由左至右為重要程度高到低。

    用例分析：拆解預約行為步驟

    先將目前系統預約行為進行用例拆解，了解完成一項預約任務的各項子行為。

    以卡片分類法，將上述整理的用例任務（藍色）進行行為分群（黃色），並整合成介面（綠色）。

    02 資訊架構設計

    根據「使用者預約行為用例分析」與「完成預約任務的用例分析」結果可得，對應介面的預約流程中，使用者最在乎的是能夠掌握可預約且符合需求的空間、時間、資訊。

    依據重要用例預約流程設計資訊架構，設計三個預約流程頁面讓使用者能夠選擇想要的時間、空間、共同使用者與獲得成功預約的訊息。

    03 介面流程

    優先選擇最重要的三個預約用例，進行介面流程設計，並分為介面（黃色矩形）、行為（藍色圓形)、系統(紫色菱形)

    用例一：使用者能夠一目瞭然看到可預約時間與空間

    首頁可讓使用者快速篩選想要的圖書館分館、空間、預約人數，並進入查看個空間可預約時段頁面。

    用例二：使用者篩選時間、空間找到符合需求的空間後，進行預約

    接續用例一，選擇完想要的時間空間，檢查是否登入後，確認預約資訊。

    用例三：使用者能夠確認預約資訊並修改

    接續用例二，使用者可確認曾經預約的資訊，並取消紀錄。

    04 任務 & 易用性測試

    我們邀請 2 位受試者，進行 low-fi prototype 測試，針對三個用例設計測試腳本，讓受訪者以放聲思考法，觀察與記錄使用者執行任務上看到的資訊、感受，以及遇到的困難。

    任務測試結果與迭代

    使用者在任務二「查看討論室頁面」時遇到最多挑戰與困難，因為介面的元件設計與資訊架構與使用者心理模型不太相符，使用上容易造成困惑，因此我們主要針對 NCCUSpace 的「查看討論室頁面」做出以下設計調整：

    1. 將空間種類下放到自由選擇的 Tab Bar，與時段表做出區隔：
      將空間種類並排，讓使用者隨意選擇，若想進一步瞭解可以點選了解空間種類差異。
    2. 日期選項改由篩選列選擇：
      將原先的空間種類換成使用者進入網站前已知資訊：預約日期，讓上面的篩選列是使用者一開始就知道要選什麼，下面的空間種類和時段由使用者當下彈性選擇。
    3. 將篩選結果重點呈現，使用教學選項刪除：
      篩選結果出現在查詢按鈕鄰近右側位置，讓使用者可以一目瞭然知道結果有幾筆選項，使用者教學在此位置意圖不明顯，故刪除，將改由首次登入時新增使用者教學，並在介面中提供略過選項。
    4. 空間時段比較表優化：
      第一版將日期與討論室放在同一層級，造成架構不清，而感到困惑，因此調整資訊呈現方式，並以結果多寡彈性呈現畫面寬度，避免畫面欄位留白。

    SUS 易用性測試結果

    SUS Score：（67.5 + 87.5）/ 2 = 79.8 （Good）
    測試結果顯示，整體介面重新設計對使用者而言，是容易理解與學習如何使用的。

    05 品牌形象

    NCCUSpace 品牌形象

    我們將原有的政大場地管理系統，以 NCCUSpace 結合政大與空間運用命名，重塑品牌形象，強調透過簡單的預約步驟，提供一個輕鬆、方便的空間預約體驗，讓學生可以更專注於高效的工作與學習。除了目前主要優化的討論室預約流程外，未來也可整合政大各處室空間預約與管理系統，方便學生、教師進行預約與管理。

    • Logo： 以書冊堆疊成建築形象，展現圖書館所提供之討論空間
    • 顏色： 結合政大自然風景與目前政大圖書館網站使用的色系，選用大地色系，給予學生舒適，且有親和力的感覺
    • 字體：選用無襯線的源樣黑體，呈現簡約俐落風格
    • 特色：以視覺化表格方式，呈現篩選時段、討論室的結果，方便使用者能夠直接選擇、查看想要預約討論室的結果

    06 UI 設計

    根據測試結果迭代 Hi-fi 原型設計，以下說明主要操作頁面

    1. 首頁

  • 下拉篩選日期、空間、使用人數
  • 清楚指引預約流程
  • 空間資訊一次掌握
  • 不需登入就可直接查詢
    •

    2. 預約頁

  • 同時比對不同可用空間
  • 顏色顯示空間狀態
  • 滑動點選預約可用時段
    •

    3. 預約確認

  • 預約資訊再次確認
  • 呈現空間展示照片
  • 新增其他使用者
    •

    4. 預約完成

  • 預約資訊提醒
  • 引導查看紀錄
    •

    5. 預約紀錄

  • 以區塊切分資訊呈現
  • 預約中：可取消紀錄與修改成員
  • 過往紀錄：了解使用狀態（報到或是未到記點情形）

    學習 Learning

    Redesign 需要從使用者的行為思考，而非僅修改介面。

    原先的介面雖然也能讓使用者成功預約討論室，但透過訪談我們了解，這些使用者都需要先經過一番摸索才能習慣現有的操作方式，因此我們將預約過程進行詳細的拆解與分析，對應訪談中獲取的洞見，重新設計新版介面，對於使用者而言，能夠幫助他們快速預約到想要的空間與時段。
