總覽 Overview

目標

為受負面情緒所困的人，設計一款 MoodLight APP ，期望藉由使用者數據，透過K-means 分群，採用NoSQL資料庫，提供客製化的情緒記錄日記與活動推薦系統，讓使用者逐漸找到生活中的快樂。

需求

  • 發展客製化改善負面情緒的方法
  • 了解大數據應用原理與流程
  • 製作行動裝置 APP 原型
    •

    原型

    問題 Problem

    在高壓的社會下，受負面情緒所困的人日益漸增，學習適當且即時地排解自我情緒，是非常重要的。

    根據2021年世界衛生組織的調查，全世界有2.8億人口患有憂鬱症，在新冠疫情的肆虐下，狀況也越發嚴重。倘若一個人的負面情緒不斷地累積且無法適當排解時，將嚴重影響身心靈健康，甚至危及性命，該如何調適負面情緒已成為當今不容忽視的問題。

    現今社會上已存在許多協助改善憂鬱情緒的方法與工具，但如何客製化且即時的抒發情緒是待解決之道。

    設計過程 Process

    01 文獻研究

    快樂方程式：快樂可以自己掌握

    正向心理學家之父馬丁（Martin Seligman）曾提出快樂方程式：

    H = S + C + V

    Enduring Level of Happiness = Set Range + Circumstances + Voluntary Control

    馬丁認為決定一個人是否能長久感到快樂的因素，有50% 來自先天的個性影響（S），10% 來自環境（C），而有 40% 則可以藉由個人的努力獲得 （V），故是否能感到快樂約有一半是源自於自身的選擇。人會感到快樂，主要是藉由做自己喜歡的事情情，如：繪畫、閱讀、運動等，這些活動刺激感官，使腦內的多巴胺或腦內啡產生，提升幸福感。

    02 流程設計

    如何讓使用者有效排解負面情緒？

    MoodLight 旨在幫助使用者透過每日的心情紀錄，協助正視自我的情緒狀況，同時蒐集每日使用數據與日記紀錄分析，讓使用者能重拾正能量，找到屬於自己的快樂方程式。

    主要提供兩大功能：

  • 快樂日記：心情愉悅時，撰寫快樂日記記錄美好心情
  • 快樂清單推薦：心情低落時，透過大數據運算，獲得客製化解決方法
    •

    03 技術運作

    人可以透過自身的行為來獲取快樂因子，因此 MoodLight 希望收集使用者的偏好數據，建立客製化推薦資料庫，並透過集群分析（K-means clustering）的方式，類聚特徵相似的使用者數據，給予最匹配的的快樂清單推薦。

    數據來源一：快樂日記比對

    根據使用者過去所填寫的快樂日記內容，推薦以前曾讓他感到愉快的活動。

    數據來源二：分群使用者推薦

    為依據和使用者擁有類似興趣的群體中，推薦會讓該群體放鬆心情的行為。

    04 介面設計

  • 風格：與美術設計合作，設計可愛溫馨的姆姆形象
  • 色調：淺藍色帶給人平靜感受
  • 流程：點選形式記錄日記，簡化步驟

    學習 Learning

    沉浸式科技的導入要能夠滿足與使用者的需求動機，才能有良好的互動成效。

    沉浸式科技的導入要能夠滿足與使用者的需求動機，才能有良好的互動成效。 經過這個專案讓我了解沉浸式科技AR、VR的使用差異，以及認識到設計師對於互動科技的選擇，對使用者的體驗有很大的影響。透過完整的測試流程，能幫助我認識使用者在操作時的行為與感受，藉此優化未來的設計。
